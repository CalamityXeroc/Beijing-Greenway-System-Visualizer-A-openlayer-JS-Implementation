## Plan: 评论区绿道化现代重设计

本次仅重做桌面评论区视觉与交互表达（不改移动端），采用“自然绿道杂志感”方向，并弱化楼层编号，改为“回复某某”的社交语义。实现方式是在不改接口契约的前提下，重构 DesktopCommentsPanel 的信息层级、视觉变量和交互反馈，使其更接近主流社媒的现代体验，同时保留现有楼中楼能力。

**Steps**
1. Phase 1: 视觉基线与变量映射
- 从现有主题变量中抽取评论区专用 token（浅色/夜色各一套），重点替换当前蓝色主导色为绿道主色系（主绿、浅绿背景、评分黄、危险红）。
- 定义评论区局部变量：容器背景、卡片背景、主文本、次文本、边框、交互高亮、点赞激活、回复提示色。
- 约束：不破坏全站其他组件变量，只在评论区作用域内落地。

2. Phase 2: 信息架构重排（*depends on 1*）
- 顶部摘要改为“社区感”头部：评分均值、讨论总量、当前排序，强化社交入口感。
- 发布区改为“社媒输入卡”：主输入更突出、操作区简化、预览区样式从“开发态盒子”改为“阅读态卡片”。
- 楼层结构弱化编号：默认展示“回复某某”，楼层编号仅保留在辅助层（必要时淡化显示）。

3. Phase 3: 评论卡片重设计（*depends on 2*）
- 评论主体采用杂志感阅读卡：提高留白、优化行高、减少硬边框压迫感。
- 用户元信息层：头像占位/昵称/时间/回复关系一行内分级展示，避免信息拥挤。
- 操作区改为社媒风格轻按钮（点赞、回复、举报、删除），统一 hover/active/disabled 反馈。
- 回复框改为嵌入式“子讨论面板”，视觉区分父评论与当前回复输入状态。

4. Phase 4: 夜间模式一致性（*parallel with 3 after base styles exist*）
- 消除现有夜间模式“偏蓝”割裂，统一为低饱和绿灰体系。
- 校正暗色下对比度：正文、次文本、边框、按钮态满足可读性与层级清晰。
- 检查夜间下点赞激活、回复标签、输入态焦点环的可见度一致。

5. Phase 5: 细节动效与状态完善（*depends on 3 and 4*）
- 补充轻量动效：卡片 hover 浮起、按钮按压反馈、回复框展开过渡。
- 完善状态态样式：加载中、空状态、错误状态、提交中按钮。
- 确保操作触达性：按钮最小点击区域、边缘场景（长评论、无评分、深层回复）。

6. Phase 6: 回归验证与微调（*depends on 5*）
- 在多条绿道详情页回归，确保评论区在不同页面宽度和滚动上下文中稳定展示。
- 校验楼中楼交互闭环：回复、取消回复、提交后刷新、点赞态回写。
- 根据视觉观感做一次小幅调参与收口（间距、字号、色彩对比）。

2. 视觉验证：亮色/夜色下分别检查头部、发布框、评论卡、回复框、操作按钮层级。

- 已确认：楼层呈现弱化编号，优先“回复某某”的社交语义。
