## 🖥️ 系统要求

| 要求 | 版本 | 说明 |
|------|------|------|
| **Node.js** | 18.x 或以上 | 后端服务 + 前端构建 |
| **npm** | 9.x 或以上 | 包管理器 |
| **PostgreSQL** | 14+ | 数据库（需要 PostGIS 扩展） |
| **PostGIS** | 3.3+ | 地理信息系统扩展库 |
| **操作系统** | Windows / macOS / Linux | 推荐 Windows 10+ |

### 快速检查安装
```bash
node --version      # 应显示 v18.x.x 或以上
npm --version       # 应显示 9.x.x 或以上
psql --version      # 应显示 PostgreSQL 14+
```

## 🔧 环境准备

### Step 1: 安装 Node.js 和 npm

**Windows:**
1. 访问 [nodejs.org](https://nodejs.org/)，下载 **LTS 版本**（18.x 或 20.x）
2. 双击安装程序，一路点 "Next"，勾选 "Add to PATH"
3. 打开 PowerShell，运行 `node --version` 验证

**macOS:**
```bash
brew install node
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt-get update
sudo apt-get install nodejs npm
```

---

### Step 2: 安装 PostgreSQL + PostGIS

#### **Windows**
1. 访问 [postgresql.org](https://www.postgresql.org/download/windows/)
2. 下载 **PostgreSQL 14+** 安装程序
3. 安装时：
   - 设置超级用户密码（记住！默认用户是 `postgres`）
   - 选择端口 `5432`（默认）
   - 勾选 "Add PostgreSQL binaries to PATH"
4. 安装后，打开 pgAdmin 4（PostgreSQL 自带 GUI）
5. 在 pgAdmin 中，右键点击 "Databases" → "Create" → "Database"，创建名为 `greenway` 的数据库
6. 在 `greenway` 数据库上右键 → "Query Tool"，执行：
   ```sql
   CREATE EXTENSION IF NOT EXISTS postgis;
   ```

#### **macOS**
```bash
brew install postgresql
brew install postgis
# 启动 PostgreSQL
brew services start postgresql

# 创建数据库
createdb greenway
psql greenway -c "CREATE EXTENSION IF NOT EXISTS postgis;"
```

#### **Linux (Ubuntu/Debian)**
```bash
sudo apt-get update
sudo apt-get install postgresql postgresql-contrib postgis

# 启动 PostgreSQL
sudo systemctl start postgresql

# 创建数据库和扩展
sudo -u postgres createdb greenway
sudo -u postgres psql greenway -c "CREATE EXTENSION IF NOT EXISTS postgis;"
```

**验证 PostGIS 已安装：**
```bash
psql -U postgres -d greenway -c "SELECT postgis_version();"
# 应输出 PostGIS 版本信息
```

---

### Step 3: 下载项目并安装依赖

假设项目位于 `C:\Users\你的用户名\Desktop\BST_lab\tryyyyyy`

```bash
cd C:\Users\你的用户名\Desktop\BST_lab\tryyyyyy

# 安装后端依赖
cd greenway-backend
npm install
cd ..

# 安装前端依赖
cd greenway-vue
npm install
cd ..
```

> **提示：** 首次安装可能需要 5-10 分钟，具体取决于网络速度

---

### Step 4: 配置环境变量

#### **后端配置** (`greenway-backend/.env.local`)

1. 复制模板文件：
   ```bash
   cd greenway-backend
   copy .env.example .env.local
   ```

2. 编辑 `.env.local`，根据你的 PostgreSQL 配置修改：
   ```env
   # 数据库配置
   DB_HOST=localhost
   DB_PORT=5432
   DB_NAME=greenway
   DB_USER=postgres
   DB_PASSWORD=你的PostgreSQL密码    # ← 改成你设置的密码
   
   # 服务器配置
   PORT=3001
   NODE_ENV=development
   ```

#### **前端配置** (`greenway-vue/.env.local`)

1. 复制模板文件：
   ```bash
   cd greenway-vue
   copy .env.example .env.local
   ```

2. 编辑 `.env.local`，填入地图 API 密钥：
   ```env
   # 高德地图 API Key（https://lbs.amap.com/）
   VITE_AMAP_KEY=你的高德API密钥
   
   # 百度地图 API Key（https://lbsyun.baidu.com/）
   VITE_BAIDU_MAP_KEY=你的百度API密钥
   
   # 后端 API 地址
   VITE_API_BASE_URL=http://localhost:3001
   ```

> **如何获取 API 密钥？**
> - 高德地图：访问 [lbs.amap.com](https://lbs.amap.com/)，登录 → 应用管理 → 创建应用 → 添加 Key
> - 百度地图：访问 [lbsyun.baidu.com](https://lbsyun.baidu.com/)，类似流程

## 🚀 一键启动

### Windows 用户（最简单）

在项目根目录，双击运行：
```
启动完整系统.bat
```

这会自动：
1. ✅ 初始化数据库（如果尚未初始化）
2. ✅ 启动后端服务（端口 3001）
3. ✅ 启动前端开发服务器（端口 5173）
4. ✅ 打开项目页面 http://localhost:5173

> **需要 2-3 秒初始化，请耐心等待**

## 📝 分步启动（手动）

如果一键启动遇到问题，可以手动分步启动：

### Terminal 1: 初始化数据库 + 启动后端

```bash
cd greenway-backend

# 初始化数据库（首次运行必须）
npm run db:init

# 启动后端服务
npm run dev
```

**预期输出：**
```
============================================================
[服务器] ✓ 后端已启动
[服务器] 地址: http://localhost:3001
[服务器] 健康检查: http://localhost:3001/health
[服务器] API 文档: http://localhost:3001/api/greenways
============================================================
```

### Terminal 2: 启动前端（新开 PowerShell 窗口）

```bash
cd greenway-vue

# 启动前端开发服务器
npm run dev
```

**预期输出：**
```
VITE v5.x.x ready in xxx ms

➜  Local:   http://localhost:5173/
➜  press h to show help
```

## ❓ 常见问题

### Q1: 启动后端时出现 `Error: connect ECONNREFUSED`

**原因：** PostgreSQL 没有启动或数据库连接参数错误

**解决方案：**
1. 检查 PostgreSQL 是否运行（Windows 任务管理器查看 `postgres.exe`）
2. 用 `psql -U postgres` 测试连接
3. 确认 `.env.local` 中的密码是否正确
4. 检查数据库 `greenway` 是否存在：
   ```bash
   psql -U postgres -l | grep greenway
   ```

### Q2: 前端无法连接后端 API

**原因：** 后端服务未启动或 `.env.local` 配置错误

**解决方案：**
1. 确认后端控制台输出了 "后端已启动" 消息
2. 访问 http://localhost:3001/health 测试后端健康状态
3. 打开浏览器开发者工具（F12）→ Network，查看 API 请求状态码

### Q3: 地图不显示瓷砖或数据

**原因：** 高德 API 密钥无效或网络连接问题

**解决方案：**
1. 检查 `greenway-vue/.env.local` 中的 `VITE_AMAP_KEY` 是否正确
2. 访问 https://lbs.amap.com/ 验证 Key 是否有效
3. 浏览器控制台（F12 → Console）查看错误信息

### Q4: 忘记了 PostgreSQL 密码

**Windows 解决方案：**
1. 按 `Win+R`，运行 `services.msc`
2. 找到 "postgresql"，右键停止服务
3. 重新安装 PostgreSQL，设置新密码

**macOS/Linux：**
```bash
sudo -u postgres psql
ALTER USER postgres PASSWORD 'new_password';
```

### Q5: 端口被占用（3001 或 5173）

**Windows：**
```bash
# 查找占用 3001 的进程
netstat -ano | findstr :3001

# 杀死进程（PID 替换为实际值）
taskkill /PID <PID> /F
```

**macOS/Linux：**
```bash
lsof -i :3001
kill -9 <PID>
```

### 核心技术栈

**前端：**
- Vue 3（框架）
- Vite（构建工具）
- OpenLayers 8（地图库）
- Pinia（状态管理）
- Vue Router（路由）

**后端：**
- Node.js（运行时）
- Express（Web 框架）
- PostgreSQL + PostGIS（数据库）
- JWT（身份认证）

**移动端：**
- Capacitor 6（跨平台框架）
- Android Studio AVD（安卓模拟器）
